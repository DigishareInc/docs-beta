Secure access to the Digishare API is mandatory. We support two primary methods of authentication depending on your integration needs. All API requests must be made over **HTTPS**; calls made over plain HTTP will fail.

::alert{type="info"}
The best way to authenticate depends on your use case. Use **PATs** for simple scripts and **OAuth 2.0** for production applications.
::

## 1. Personal Access Token (PAT)

Personal Access Tokens are essentially passwords for your account that you can use to authenticate with the API. They are ideal for quick integrations or when you don't need to implement a full OAuth flow.

::steps

### Generate your Token

  Navigate to **Advanced Settings > API Credentials** or use the direct link: [https://app.digishare.ma/api-credentials](https://app.digishare.ma/api-credentials).

### Select Scopes

  Choose the specific permissions this token should have. We recommend the "Principle of Least Privilege"—only grant what you actually need.

### Copy & Secure

  Copy the token immediately. For security reasons, it will **not** be shown again. If you lose it, you will need to revoke it and generate a new one.
::

### Authorization Header Example

Include the token in the `Authorization` header as a `Bearer` token.

::code-group

  ```bash [cURL]
  curl -X GET "https://api.digishare.ma/v1/user/profile" \
    -H "Authorization: Bearer YOUR_TOKEN_HERE" \
    -H "Accept: application/json"
  ```

  ```javascript [Axios]
  import axios from 'axios';
  
  const response = await axios.get('https://api.digishare.ma/v1/user/profile', {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN_HERE'
    }
  });
  ```

  ```javascript [Fetch]
  const response = await fetch('https://api.digishare.ma/v1/user/profile', {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN_HERE',
      'Accept': 'application/json'
    }
  });
  const data = await response.json();
  ```

## 2. OAuth 2.0 Client Credentials

Use the **Client Credentials Grant** for app-to-app communication where no user intervention is required. This is the most secure method for production applications.

### Flow Overview

1. **Exchange** your `Client ID` and `Client Secret` for an access token.
2. **Retrieve** the token from the response.
3. **Use** the token in the `Authorization` header for all API calls.

### Token Request

Send a `POST` request to the `/oauth/token` endpoint.

::code-group

  ```bash [Request]
  curl -X POST "https://api.digishare.ma/v1/oauth/token" \
    -H "Content-Type: application/json" \
    -d '{
      "grant_type": "client_credentials",
      "client_id": "YOUR_CLIENT_ID",
      "client_secret": "YOUR_CLIENT_SECRET"
    }'
  ```

  ```json [Response]
  {
    "token_type": "Bearer",
    "expires_in": 31536000,
    "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiIsIm..."
  }
  ```

## Security & Best Practices

::alert{type="warning"}
**Never expose credentials** in client-side code (browsers/mobile). These should only be handled in secure server-side environments.
::

* **Rotation**: Regularly rotate your Personal Access Tokens and Client Secrets.
* **Revocation**: If a token is compromised, revoke it immediately via the dashboard.
* **Scopes**: Always limit the scope of tokens to the specific resources they need to access.
* **Rate Limits**: All authentication methods are subject to our API rate limits.
