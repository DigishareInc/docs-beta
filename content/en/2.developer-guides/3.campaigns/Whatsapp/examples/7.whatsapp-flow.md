This guide demonstrates how to orchestrate WhatsApp campaigns featuring **Flows** for streamlined data collection and interactive service paths.

::note
**Prerequisites**:
- Valid **Bearer Token** in Authorization header.
- **Digishare Message Template** (**WABA TM** type) configured with an active Meta Flow.
- **Flow ID** and **Start Screen ID** configured in your Dashboard.
::

## 1. Flow Operational Logic

Flows empower you to deploy structured forms and multi-step interactive screens directly within the WhatsApp interface. Initial data can be passed to the Flow using the `flow_token` and `flow_action_payload` parameters.

## 2. API Implementation

Personalized Flow parameters must be bridged through the `global_data` block to ensure they are available for each recipient's unique session.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send a campaign featuring an interactive WhatsApp Flow.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Table Reservation Flow
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    flow_token: "%recipient.flow_token%"
    flow_action_payload: "%recipient.flow_action_payload%"
  recipients:
    - wa_id: "212600000000"
      flow_token: "res_token_789"
      flow_action_payload:
        screen: "START_SCREEN"
        data:
          init_val: "123"
  tags: []
  conversation_id: "message-out"
---
::

::important
WhatsApp Flows are a high-fidelity feature restricted to official **WABA TM** templates. Ensure your blueprint is approved by Meta before deployment.
::
