This guide details the orchestration of WhatsApp campaigns featuring **External Link** buttons for seamless navigation to web properties.

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template** (ITM or **WABA TM**) with a `URL` button component enabled.
::

## 1. Governance & Link Architecture

Digishare supports two primary URL architectures within your message assets:
- **Static**: A fixed destination URL shared across all campaign recipients.
- **Dynamic**: A base URL appended with a variable suffix (e.g., `https://site.ma/orders/{{button0}}`).

## 2. API Implementation

For dynamic URL orchestration, the variable suffix must be explicitly bridged through the `global_data` block.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send a campaign featuring dynamic URL redirection suffixes.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Order Payment Orchestration
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    button0: "%recipient.button0%"
  recipients:
    - wa_id: "212600000000"
      button0: "ORD-789-XP"
---
::

::warning
The base URL is defined during the blueprint phase in your Dashboard. The API provides the **bridging mechanism** for the dynamic suffix only.
::
