This guide details the orchestration of WhatsApp campaigns featuring **Carousels**—a high-fidelity format supporting up to 10 interactive sliding cards.

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template** (**WABA TM** type) approved for Carousel orchestration.
::

## 1. Carousel Card Governance

Each card within a carousel supports unique media (image/video) and specific body variables for deep personalization.

### Technical Attributes
- `header_url`: The visual asset (image/video) for each specific card.
- `body_variables`: Localized array of values for card-specific variable injection.

## 2. API Implementation

For dynamic carousels, the `cards` array must be mapped within the `global_data` block to ensure distinct personalization for each recipient.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send a personalized product carousel showcasing unique cards per recipient.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Seasonal Product Showcase
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    cards: "%recipient.cards%"
  recipients:
    - wa_id: "212600000000"
      cards:
        - header_url: "https://brand.ma/p1.jpg"
          body_variables: ["Vastux Pro", "199 USD"]
        - header_url: "https://brand.ma/p2.jpg"
          body_variables: ["Vastux Air", "99 USD"]
---
::

::important
Carousel orchestration is exclusively available for **Marketing** classification **WABA TM** templates.
::
