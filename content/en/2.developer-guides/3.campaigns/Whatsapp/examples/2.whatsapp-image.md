This guide demonstrates how to orchestrate a WhatsApp campaign featuring **high-fidelity images**.

::note
**Prerequisites**:
- Valid **Bearer Token** in Authorization header.
- **Digishare Message Template** (ITM or **WABA TM**) with an image header component enabled.
- Publicly accessible image URL (JPG or PNG).
::

## 1. Payload Requirements

For personalized images, provide the header link and recipient information in the `global_data` and `recipients` blocks.

### Required Fields
- `wa_id`: Recipient's phone number (in `global_data` as template variable).
- `header.link`: Direct public URL of the image (JPG or PNG).
- `tags`: Optional array for categorizing campaigns.
- `conversation_id`: Optional identifier for conversation tracking.

## 2. API Implementation

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send unique promotional images to each specific recipient.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Product Promo Campaign
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    header:
      link: "https://cdn.brand.ma/promo-blue.jpg"
  recipients:
    - wa_id: "212600000000"
    - wa_id: "212600000001"
  tags: []
  conversation_id: "message-out"
---
::

## 3. Operational Best Practices

- **Accessibility**: Ensure your image URLs are publicly reachable without authentication.
- **Aspect Ratio**: A **1:1 square ratio** is recommended for optimal rendering in WhatsApp chat lists and carousels.
