This guide demonstrates how to orchestrate WhatsApp campaigns featuring **OTP (One-Time Password)** codes for secure secondary authentication.

::note
**Prerequisites**:
- Valid **Bearer Token** in Authorization header.
- **Digishare Message Template** (**WABA TM** type) approved under the **Authentication** category.
::

## 1. OTP Governance

Official WhatsApp OTP templates support native platform features like auto-complete and "copy code" buttons. For these templates, the `otp_code` field is a reserved technical identifier.

## 2. API Implementation

Even for reserved fields, dynamic values must be explicitly bridged through the `global_data` block for individual recipients.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send a high-security OTP verification campaign.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Security Verification Outreach
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    otp_code: "%recipient.otp_code%"
  recipients:
    - wa_id: "212600000000"
      otp_code: "456-789"
  tags: []
  conversation_id: "message-out"
---
::

::important
The `otp_code` field is reserved for official **Authentication** WABA TM templates. For standard text templates, utilize the generalized variable mapping logic (e.g., `{{1}}`).
::
