This guide explains how to send a WhatsApp campaign including a **PDF document**.

::note

**Prerequisites**:
- Valid **Bearer Token** in Authorization header.
- **Digishare Message Template** (ITM or **WABA TM**) configured to accept a header with PDF link.
- Publicly accessible PDF URL.
::

## 1. Payload Requirements

To send a personalized PDF, you must provide the header link and recipient information in the `global_data` and `recipients` blocks.

### Required Fields
- `wa_id`: Recipient's phone number (in `global_data` as template variable).
- `header.link`: Publicly accessible URL of the PDF.
- `tags`: Optional array for categorizing campaigns.
- `conversation_id`: Optional identifier for conversation tracking.

## 2. API Implementation

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send a personalized PDF document to multiple recipients.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Invoice Distribution
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    header:
      link: "https://your-site.com/inv_january.pdf"
  recipients:
    - wa_id: "212600000000"
    - wa_id: "212600000001"
  tags:
    - "campaign"
  conversation_id: "message-out"
---
::

## 3. Handling Static PDFs

If your campaign uses the same PDF for all recipients, define the header link once in the `global_data` block:

```json
{
  "title": "Monthly Catalog",
  "channel": "whatsapp",
  "sender_label_id": "your-sender-label-id",
  "message_template_id": "your-message-template-id",
  "global_data": { 
    "wa_id": "%recipient.wa_id%",
    "header": {
      "link": "https://your-site.com/catalog.pdf"
    }
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ],
  "tags": ["campaign"]
}
```
