This guide explains how to send a WhatsApp campaign including a **PDF document**.

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template** (ITM or **WABA TM**) configured to accept a PDF header.
::

## 1. Payload Requirements

To send a personalized PDF, you must bridge the `header_url` and `filename` variables via the `global_data` block.

### Required Logic
- `wa_id`: Recipient's phone number.
- `header_url`: Publicly accessible URL of the PDF.
- `filename`: The display name of the file on WhatsApp.

## 2. API Implementation

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Send a personalized PDF document to multiple recipients.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Invoice Distribution
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    header_url: "%recipient.header_url%"
    filename: "%recipient.filename%"
  recipients:
    - wa_id: "212600000000"
      header_url: "https://your-site.com/inv_january.pdf"
      filename: "invoice_january.pdf"
    - wa_id: "212600000001"
      header_url: "https://your-site.com/inv_february.pdf"
      filename: "invoice_february.pdf"
---
::

## 3. Handling Static PDFs

If your campaign uses the same PDF for all recipients, define the values once in the `global_data` block:

```json
{
  "title": "Monthly Catalog",
  "global_data": { 
    "header_url": "https://your-site.com/catalog.pdf", 
    "filename": "Product_Catalog.pdf" 
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ]
}
```
