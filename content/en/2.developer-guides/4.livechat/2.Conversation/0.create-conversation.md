Create a new conversation with a contact via an API provider. The system will intelligently resolve the contact and their platform identity, and manage any existing active or archived conversations according to the flags you provide.

**Endpoint**: `POST https://api.digishare.ma/v1/conversations/provider`

### Key Parameters / Payload

| Parameter                          | Type    | Required | Description                                                                            |
| :--------------------------------- | :------ | :------- | :------------------------------------------------------------------------------------- |
| `provider_instance_id`             | String  | Yes      | The ID of the API provider instance handling the channel.                              |
| `channel`                          | String  | Yes      | The messaging channel (e.g. `whatsapp`, `sms`).                                        |
| `recipient_id`                     | String  | Yes      | The phone number, WhatsApp ID, or platform-specific identifier of the contact.         |
| `platform_link_id`                 | String  | No       | ID of an existing PlatformLink, if known. Skips contact resolution if provided.        |
| `archive_active_conversation`      | Boolean | No       | Default: `true`. Archives the current active conversation before creating a new one.   |
| `reactivate_archived_conversation` | Boolean | No       | Default: `false`. Reactivates the most recently archived conversation.                 |

::api-playground
---
method: POST
url: "https://api.digishare.ma/v1/conversations/provider"
headers:
  Authorization: "Bearer <JWT_TOKEN>"
  Content-Type: "application/json"
  selectedcompanyid: "<COMPANY_ID>"
body:
  provider_instance_id: "<PROVIDER_INSTANCE_ID>"
  channel: "whatsapp"
  archive_active_conversation: true
  reactivate_archived_conversation: false
  recipient_id: "<RECIPIENT_PHONE_NUMBER>"
  third_id: "<THIRD_ID>"
responseSample:
  id: "conv_123"
  status: "active"
  platform_link_id: "pl_456"
---
::

### Execution Flow

1. **Validation**: Verifies the `provider_instance_id` exists.
2. **Contact Resolution**: Resolves the `PlatformLink`. If not found, creates a minimal `Third` contact record and a `PlatformLink` on the fly.
3. **Reactivation Check**: If `reactivate_archived_conversation` is `true`, the last archived conversation is reactivated. The currently active one is archived first if `archive_active_conversation` is also `true`.
4. **Archive Check**: If an active conversation exists and reactivation is not requested, archives it when `archive_active_conversation` is `true`.
5. **Creation**: Dispatches `CreateConversationAction` to create a new active conversation with the resolved template and contact.
