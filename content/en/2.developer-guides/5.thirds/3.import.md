Bulk import contacts from CSV or Excel files. The import is processed asynchronously via a queue.

---

## Import Endpoint

::api-playground
---
method: POST
url: "https://api.digishare.ma/v1/thirds/import"
headers:
  Authorization: "Bearer YOUR_ACCESS_TOKEN"
  Content-Type: "application/json"
body:
  data:
    - first_name: "John"
      last_name: "Doe"
      email: "john@example.com"
      cf_balance: 150.75
responseSample:
  status: "queued"
---
::

## Ingestion API (Egressor)

For high-volume event ingestion, you can use the Egressor API. This is recommended for external systems sending automated updates.

::api-playground
---
method: POST
url: "https://api-app.digishare.ma/v1/event/import_thirds"
headers:
  Authorization: "Bearer YOUR_ACCESS_TOKEN"
  Content-Type: "application/json"
body:
  data:
    - first_name: "John"
      last_name: "Doe"
      email: "john@example.com"
responseSample:
  event_id: "DS_550e8400_e29b_..."
  status: "accepted"
---
::

### Comparison

| Feature | Standard API | Ingestion API |
|---------|--------------|---------------|
| **Endpoint** | `/v1/thirds/import` | `/v1/event/import_thirds` |
| **Max Batch** | Spreadsheet size | 1,000 items |
| **Response** | `204 No Content` | `202 Accepted` |
| **Reliability** | Synchronous Validation | Kafka-backed Queue |
| **Use Case** | Manual Uploads | System Integrations |

## Request Body

The import request accepts either a base64 file or a raw JSON array.

### Parameters

- **`file`** (string, optional)
  The base64-encoded content of your CSV or XLSX file.
- **`data`** (array, optional)
  A JSON array of contact objects. This is ideal for programmatic integrations.
- **`company_id`** (string, optional)
  The ID of the company to import data into.

::alert{type="warning" icon="i-heroicons-exclamation-triangle"}
**Constraint:** You must provide exactly one of `file` or `data` in your request.
::

::alert{type="info" icon="i-heroicons-information-circle"}
**Automatic Assignment:** The `company_id` is automatically extracted from your auth token. You only need to provide it if you have permission to act on behalf of another company.
::

### Response

Returns `204 No Content` immediately. The import is processed in the background.

## File Format

Your spreadsheet must include a header row. Here are the supported columns:

### Standard Columns

| Column | Description |
|--------|-------------|
| `id` | Existing third ID (for updates) |
| `ext_ref` | External reference (for matching) |
| `first_name` | First name |
| `last_name` | Last name |
| `name` | Full name |
| `email` | Email address |
| `mobile` | Mobile phone |
| `phone` | Landline phone |
| `wa_id` | WhatsApp ID |
| `address` | Address |
| `status` | Active status (true/false) |
| `premium` | Premium flag (true/false) |
| `verified` | Verified flag (true/false) |
| `is_company` | Company flag (true/false) |
| `tags` | Tags (semicolon-separated: `vip;customer;lead`) |
| `wa_ids` | Additional WhatsApp IDs (semicolon-separated) |
| `city_id` | City ID (hashed) |
| `country_id` | Country ID (hashed) |
| `state_id` | State ID (hashed) |
| `source_id` | Source ID (hashed) |

### Custom Field Columns

Add custom fields by prefixing column names with `cf_` or `custom_`:

| Column | Stored As |
|--------|-----------|
| `cf_balance` | `balance` |
| `cf_loyalty_tier` | `loyalty_tier` |
| `custom_is_vip` | `is_vip` |

## Import Behavior

### Matching Existing Records

The import automatically matches existing records by:

1. **`id`** - If provided, updates the existing record
2. **`ext_ref`** - Matches by external reference within the company
3. **`wa_id`** - Matches by WhatsApp ID within the company
4. **`mobile` / `phone`** - Matches by standardized phone number (automatically converts `0...` to `212...` before lookup)

If no match is found, a new record is created.

### Custom Fields Sync

Custom fields (columns with `cf_` or `custom_` prefix) are:

- Automatically synced to the Custom Fields database
- Type-inferred from values:
  - `true`, `yes`, `1`, `oui` â†’ boolean `true`
  - `false`, `no`, `0`, `non` â†’ boolean `false`
  - Numbers with decimals â†’ `float`
  - Whole numbers â†’ `int`
  - Other â†’ `string`
- Field definitions are auto-created if they don't exist

::tip
**Filterable after import.** Once imported, you can filter contacts by custom fields using the `custom_filters` parameter.
::

## Error Handling

| Status | Description |
|--------|-------------|
| `204` | Import queued successfully |
| `400` | Invalid file format |
| `401` | Unauthorized |
| `422` | Validation error (file required) |

::note
**Async Processing.** The import runs in the background. Large files may take several minutes to process completely.
::
