This guide explains how to send WhatsApp campaigns using **Digishare Message Templates**. These templates can be **Text**, **ITM** (Interactive Digishare), or **WABA TM** (Meta-approved Template).

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template ID** (created in your Digishare Dashboard).
- **Sender Label ID** (e.g., `YOUR_SENDER_ID`) representing your WhatsApp channel.
::

## 1. Payload Structure (Modern Mode)

We recommend using the **flattened** structure for better readability and simplified maintenance of your integrations.

### Configuration Fields

- `title`: Internal campaign name.
- `channel`: Channel type (`whatsapp`).
- `sender_label_id`: ID of the sender channel.
- `message_template_id`: ID of your **Digishare Message Template** (Text, ITM, or WABA).
- `conversation_id`: (Optional) ID of an existing conversation or an integration trigger:
  - `message-out`: Integrate immediately upon delivery.
  - `replay-button`: Integrate only after a button click.
  - `any-reply`: Integrate upon any user response.
  - `conversed`: Integrate upon engagement.
- `conversation_tags`: (Optional) Array of tags to apply to the generated conversation.
- `recipients`: Recipient list with their specific data.
- `global_data`: (Optional) Shared data or dynamic mapping.

## 2. Advanced Scheduling

Control exactly when and how your campaign is sent using specialized date fields.

| Field | Description | Post-Campaign Response |
| :--- | :--- | :--- |
| `start_datetime` | The earliest date/time to begin sending. | N/A |
| `end_datetime` | (Coming Soon) Stops sending **new** campaign messages, but **reactions to client feedback** (buttons, flows) remain active. | **Active** |
| `terminate_datetime` | (Coming Soon) Absolute hard stop for the campaign **and all interactive reactions**. No more processing. | **Inactive** |

### Why use both `end` and `terminate`?
- **`end_datetime`**: Professional stop for the "blast." Use this when you want to stop starting new threads but still allow users who received the message to interact with buttons or flows.
- **`terminate_datetime`**: Final "Kill-switch." Use this for **time-critical offers**. Once reached, the campaign is fully abandoned, and any user clicking a button will not trigger a reaction.

## 3. Example: Personalized Order Notification

If your template uses variables like `Hello {{name}}, your order {{order_id}} is ready.`, the optimal payload is:

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Send a personalized WhatsApp campaign." :variables='{"token": "YOUR_TOKEN", "senderId": "YOUR_SENDER_ID", "templateId": "YOUR_TEMPLATE_ID"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Order Notification", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "name": "John", "order_id": "ORD-123"}, {"wa_id": "212600000001", "name": "Jane", "order_id": "ORD-456"}] }'}
::

## 4. Dynamic Mapping with `global_data`

The `global_data` block allows you to avoid data repetition or create complex links by mapping global variables to recipient attributes using the `%recipient.field%` syntax.

### Use Case: Personalized Tracking Links

Instead of sending the full URL for each recipient, you can define a global pattern:

```json
{
  "global_data": { 
    "company_name": "Digishare", 
    "tracking_link": "https://orders.ma/%recipient.order_id%" 
  },
  "recipients": [
    { "wa_id": "2126...", "name": "John", "order_id": "123" },
    { "wa_id": "2126...", "name": "Jane", "order_id": "456" }
  ]
}
```

> [!TIP]
> In this example, the first recipient will receive a link pointing to `https://orders.ma/123`.

## 5. Conversation IDs & Tags

Automatically organize your business threads by linking campaigns to existing conversations or applying classification tags.

```json
{
  "title": "Lead Gen Campaign",
  "conversation_id": "conv_987654321",
  "conversation_tags": ["new_lead", "summer_promo"],
  "channel": "whatsapp",
  "recipients": [ 
    { "wa_id": "212600000000", "name": "John" }
  ]
}
```

::important
Tags are applied to the conversation **as soon as the user responds** to the initial campaign message or when the message is successfully delivered (depending on your channel settings).
::
