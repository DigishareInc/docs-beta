This guide explains how to send a WhatsApp campaign including an **external link button**.

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template** (ITM or **WABA TM**) containing a `URL` type button.
::

## 1. Link Types

Digishare supports two types of URLs in your templates:
- **Static**: The same for all recipients.
- **Dynamic**: The template contains a variable at the end of the URL (e.g., `https://site.com/id/{{1}}`).

## 2. Structure of Payload

For a dynamic URL, pass the variable value in the `button0` field.

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Send a dynamic URL button." :variables='{"token": "YOUR_TOKEN", "senderId": "YOUR_SENDER_ID", "templateId": "YOUR_TEMPLATE_ID"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Order Payment", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "button0": "ORD-789"}] }'}
::

::warning
The base URL is defined during template creation in your Dashboard. The API only allows sending the **dynamic suffix** (the variable).
::
