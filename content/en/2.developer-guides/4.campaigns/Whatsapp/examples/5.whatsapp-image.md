This guide shows how to send a WhatsApp campaign including an **image**.

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template** (ITM or **WABA TM**) configured to accept an image.
::

## 1. Payload Preparation

To send an image, you must provide the public URL of the image (JPG or PNG) in the `header_url` field.

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Send a personalized image." :variables='{"token": "YOUR_TOKEN", "senderId": "YOUR_SENDER_ID", "templateId": "YOUR_TEMPLATE_ID"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Product Promo", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "header_url": "https://your-site.com/product-123.jpg"}] }'}
::

## 2. Using `global_data` for the image

If the image is the same for everyone, use `global_data` to simplify your payload and avoid repetition.

```json
{
  "title": "Summer Sales Announcement",
  "channel": "whatsapp",
  "sender_label_id": "YOUR_SENDER_ID",
  "message_template_id": "YOUR_TEMPLATE_ID",
  "global_data": { 
    "header_url": "https://site.com/summer_sales.png" 
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ]
}
```

## 3. Best Practices

| Field | Description |
| :--- | :--- |
| `header_url` | Public image URL (JPG/PNG). Must be direct link. |

::warning
Make sure the image URL is **publicly accessible**. Test it in an incognito browser before sending. If the image is behind a login, delivery will fail.
::
