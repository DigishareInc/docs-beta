This guide explains how to send a WhatsApp campaign including a **PDF document**.

::note
**Prerequisites**:
- Valid **Bearer Token**.
- **Digishare Message Template** (ITM or **WABA TM**) configured to accept a PDF.
::

## 1. Payload Preparation

To send a PDF, you must provide the public URL of the file in the `header_url` field.

### Required Fields
- `wa_id`: Recipient's number.
- `header_url`: Direct URL of the PDF file.
- `filename`: (Optional) Custom file name displayed on WhatsApp.

## 2. Payload Example

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Send a personalized PDF." :variables='{"token": "YOUR_TOKEN", "senderId": "YOUR_SENDER_ID", "templateId": "YOUR_TEMPLATE_ID"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Invoice Sending", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "header_url": "https://your-site.com/invoice-123.pdf", "filename": "invoice_january.pdf"}] }'}
::

## 3. Using `global_data` for the PDF

If the PDF is the same for everyone (e.g., a catalog), place it in `global_data`.

```json
{
  "title": "Summer Catalog",
  "global_data": { 
    "header_url": "https://cms.com/catalog.pdf", 
    "filename": "Digishare_Catalog.pdf" 
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ]
}
```
