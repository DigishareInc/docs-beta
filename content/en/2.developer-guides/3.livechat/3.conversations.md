This document explains the core functionalities for managing and interacting with conversations via API. It covers checking for existing conversations, and sending messages to them.

## 1. Check Conversation

Query the system to see if a conversation already exists for a specific set of parameters. This helps retrieve the current state of a conversation and prevents duplicates.

**Endpoint**: `GET https://calclytics.digishare.ma/api/conversations/check`

### Query Parameters

All IDs should be provided in their encoded format.

| Parameter           | Type   | Description                                           |
| :------------------ | :----- | :---------------------------------------------------- |
| `company_id`        | String | Your Company Encoded ID.                              |
| `api_provider_id`   | String | The Encoded API Provider Instance ID.                 |
| `channel`           | String | The messaging channel (e.g. `whatsapp`, `sms`).       |
| `third_id`          | String | The encoded ID of the Third (contact).                |
| `channel_identity`  | String | The identity associated with the channel.             |
| `platform_link_id`  | String | The encoded PlatformLink ID *(Optional)*.             |

### Check Request Example

```bash
curl 'https://calclytics.digishare.ma/api/conversations/check?company_id=780ew4kx3kdyjpa5&api_provider_id=zryw5xlrr8l06bqa&channel=whatsapp&third_id=0qwylwge5bvmpz65&channel_identity=212777786877' \
  -H 'sec-ch-ua-platform: "Windows"' \
  -H 'Authorization: Bearer <JWT_TOKEN>' \
  -H 'Referer: https://app.digishare.ma/' \
  -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)' \
  -H 'Accept: application/json, text/plain, */*' \
  -H 'sec-ch-ua-mobile: ?0'
```

### Check Response

Returns a JSON object detailing the discovery:

```json
{
  "success": true,
  "found": true, 
  "conversation": {
    "id": "...",
    "is_active": true,
    "last_message": { ... },
    "created_at": "..."
  },
  "identity_owner": "...",
  "timestamp": "2026-03-03T12:00:00Z"
}
```

## 2. Send Message

Send a message from an Agent (system user) to the contact in a conversation.

**Endpoint**: `POST https://api-app.digishare.ma/v1/event/conversation_message`

### Key Parameters / Payload

| Field                      | Type    | Description                                                                              |
| :------------------------- | :------ | :--------------------------------------------------------------------------------------- |
| `event_type`               | String  | Should be set to `conversation_message`.                                                 |
| `channel`                  | String  | The messaging channel (e.g. `whatsapp`).                                                 |
| `type`                     | String  | The type of the message (e.g. `text`).                                                   |
| `send_to_third`            | Boolean | Set to `true` to send the message to the external platform provider.                     |
| `api_provider_instance_id` | String  | The required ID of the API provider instance.                                            |
| `recipient_id`             | String  | The specific identifier of the recipient.                                                |
| `body`                     | String  | The actual content of the message.                                                       |
| `conversation_id`          | String  | Unique ID for the conversation *(Optional)*.                                             |

::warning
**24-Hour Guard**: If the elapsed time since the last inbound message exceeds 24 hours (e.g., WhatsApp's customer service window), delivery may be restricted by your provider.
::

### Message Request Example

```bash
curl --location 'https://api-app.digishare.ma/v1/event/conversation_message' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <JWT_TOKEN>' \
--data '{
    "event_type": "conversation_message",
    "channel": "whatsapp",
    "type": "text",
    "send_to_third": true,
    "api_provider_instance_id": "6z70eykz3zmb84qo",
    "recipient_id": "212653565591",
    "body": "Message Body"
  }'
```

### Message Response (Success)

```json
{
    "event_id": "2633736349319958528",
    "status": "accepted",
    "timestamp": "2026-02-12T14:22:23.133414141Z"
}
```
