This document explains the core functionalities for checking if a conversation already exists for a specific set of parameters.


Query the system to see if a conversation already exists for a specific set of parameters. This helps retrieve the current state of a conversation and prevents duplicates.

**Endpoint**: `GET https://calclytics.digishare.ma/api/conversations/check`

### Query Parameters

All IDs should be provided in their encoded format.

| Parameter           | Type   | Description                                           |
| :------------------ | :----- | :---------------------------------------------------- |
| `company_id`        | String | Your Company Encoded ID.                              |
| `api_provider_id`   | String | The Encoded API Provider Instance ID.                 |
| `channel`           | String | The messaging channel (e.g. `whatsapp`, `sms`).       |
| `third_id`          | String | The encoded ID of the Third (contact).                |
| `channel_identity`  | String | The identity associated with the channel.             |
| `platform_link_id`  | String | The encoded PlatformLink ID *(Optional)*.             |

### Check Request Example

```bash
curl 'https://calclytics.digishare.ma/api/conversations/check?company_id=780ew4kx3kdyjpa5&api_provider_id=zryw5xlrr8l06bqa&channel=whatsapp&third_id=0qwylwge5bvmpz65&channel_identity=212777786877' \
  -H 'sec-ch-ua-platform: "Windows"' \
  -H 'Authorization: Bearer <JWT_TOKEN>' \
  -H 'Referer: https://app.digishare.ma/' \
  -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)' \
  -H 'Accept: application/json, text/plain, */*' \
  -H 'sec-ch-ua-mobile: ?0'
```

### Check Response

Returns a JSON object detailing the discovery:

```json
{
  "success": true,
  "found": true, 
  "conversation": {
    "id": "...",
    "is_active": true,
    "last_message": { ... },
    "created_at": "..."
  },
  "identity_owner": "...",
  "timestamp": "2026-03-03T12:00:00Z"
}
```
