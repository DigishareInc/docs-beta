This document outlines the workflow for integrating an **eCommerce application** with the Digishare Platform. This enables automated customer support and order notifications.

::note
**Prerequisite**: Ensure you have authenticated your client. See [Authentication](/developer-guides/authentication).
::

## 1. Automation Lifecycle

::steps

### Media Assets Preparation

(Optional) Upload necessary documents or media (e.g., invoices, product images) to retrieve persistent IDs for your automated messages.

### Ticket & Campaign Entry

Your e-commerce application creates a ticket when an order event occurs or a customer requires assistance.

### Automated Message Dispatch

The Digishare Platform automatically triggers a personalized WhatsApp notification based on your predefined templates.

### Real-time Status Sync

Digishare synchronizes with your application via webhooks to provide live updates on distribution and engagement.
::

## 2. Media Assets Preparation

**Endpoint**: `POST https://api.digishare.ma/v1/messagesTemplates`

Before creating a ticket with an attachment (like an invoice PDF), you must upload the file to Digishare to get a `media_id`.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/messagesTemplates
description: Upload a file (PDF, image) to get a media_id for use in tickets.
variables:
  token: YOUR_TOKEN
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  type: file
  category: document
  file:
    base64: "data:application/pdf;base64,JVBERi0xLjQKJ..."
    file_name: invoice
    mime_type: application/pdf
    extension: pdf
---
::

> **Response**: Save the returned `data.id` (e.g., `YOUR_MEDIA_ID`). Use this as the `PRE_UPLOADED_HASHED_ID` in the next step.

---

## 3. Ticket & Campaign Orchestration

**Endpoint**: `POST https://api.digishare.ma/v1/tickets`

Initiate the support or notification flow by creating a ticket. You can map external references (such as Order IDs) to link your system's data with Digishare.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/tickets
description: Create a support ticket for order confirmation or customer issues.
variables:
  token: YOUR_TOKEN
  mediaId: PRE_UPLOADED_HASHED_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  external_id: "ORDER-12345"
  type_ticket_id: npvrxamq9l94y8dz
  ticket_status_id: x9q3wrlv0m7vzea8
  create_third_data: true
  information:
    third:
      ext_ref: "CUSTOMER-67890"
      name: Ahmed Alami
      wa_id: "212600000000"
      email: ahmed.alami@example.com
    campaign_data:
      order_id: "12345"
      invoice_document:
        provider: digishare
        id: "{mediaId}"
      items_count: "5"
      order_total: "500.00"
      delivery_address: "Casablanca, Bd Zerktouni"
  subject: Order Confirmation
  lang: fr
---
::

::tip
Use `campaign_data` to pass dynamic variables that can be used in your WhatsApp templates.
::

## 4. Real-time Status Sync

Configure your system to listen for `ticket.updated` events.

**Event**: `ticket.updated`

```json
{
  "event": "ticket.updated",
  "data": {
    "id": "hashed_ticket_id",
    "data": {
      "id": "hashed_ticket_id",
      "external_id": "ORDER-12345",
      "subject": "Order Confirmation",
      "ticket_number": "342106",
      "type_ticket_id": "npvrxamq9l94y8dz",
      "third_id": "hashed_third_id",
      "ticket_status_id": "yqdwrbka7kz69v8e",
      "information": {
        "third": {
          "name": "Ahmed Alami",
          "email": "ahmed.alami@example.com",
          "wa_id": "212600000000",
          "ext_ref": "CUSTOMER-67890"
        },
        "campaign_data": {
          "order_id": "12345"
        }
      }
    },
    "changes": {
      "ticket_status_id": "yqdwrbka7kz69v8e"
    }
  }
}
```

::tip
The `ticket_status_id`: `yqdwrbka7kz69v8e` corresponds to the status **Order Confirmed**.
::
