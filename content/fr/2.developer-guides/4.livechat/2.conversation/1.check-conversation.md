Ce document explique comment vérifier l'existence des conversations.


Interrogez le système pour voir si une conversation existe déjà pour des paramètres spécifiques. Cela permet de récupérer l'état actuel d'une conversation et d'éviter les doublons.

**Endpoint** : `GET https://calclytics.digishare.ma/api/conversations/check`

### Paramètres de la Requête

Tous les identifiants (ID) doivent être fournis dans leur format encodé.

| Paramètre           | Type   | Description                                           |
| :------------------ | :----- | :---------------------------------------------------- |
| `company_id`        | String | L'ID encodé de votre entreprise.                      |
| `api_provider_id`   | String | L'ID encodé de l'instance API du fournisseur.         |
| `channel`           | String | Le canal de messagerie (ex : `whatsapp`, `sms`).      |
| `third_id`          | String | L'ID encodé du Contact (Third).                       |
| `channel_identity`  | String | L'identité associée au canal.                         |
| `platform_link_id`  | String | L'ID encodé du PlatformLink *(Optionnel)*.            |

::api-playground
---
method: GET
url: "https://api.digishare.ma/v1/conversations/check"
headers:
  Authorization: "Bearer <JWT_TOKEN>"
variables:
  company_id: "780ew4kx3kdyjpa5"
  api_provider_id: "zryw5xlrr8l06bqa"
  channel: "whatsapp"
  third_id: "0qwylwge5bvmpz65"
  channel_identity: "212777786877"
responseSample:
  success: true
  found: true
  conversation:
    id: "conv_123"
    is_active: true
    last_message:
      text: "Bonjour"
    created_at: "2026-03-03T12:00:00Z"
  identity_owner: "pl_456"
---
