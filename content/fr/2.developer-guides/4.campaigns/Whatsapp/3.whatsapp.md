Ce guide explique comment envoyer des campagnes WhatsApp en utilisant les **Modèles de Message Digishare**. Ces modèles peuvent être de type **Texte**, **ITM** (Interactif Digishare) ou **WABA TM** (Modèle approuvé par Meta).

::note
**Prérequis** :
- Un **Bearer Token** valide.
- L'ID du **Modèle de Message Digishare** (créé dans votre tableau de bord Digishare).
- Un **ID de Label d'Expéditeur** (ex: `VOTRE_SENDER_ID`) représentant votre canal WhatsApp.
::

## 1. Structure de la Charge Utile (Payload Moderne)

Nous recommandons d'utiliser la structure **aplatie** pour une meilleure lisibilité et une maintenance simplifiée de vos intégrations.

### Champs de Configuration

- `title`: Nom interne de la campagne.
- `channel`: Type de canal (`whatsapp`).
- `sender_label_id`: ID du canal d'expédition.
- `message_template_id`: ID de votre **Modèle de Message Digishare** (Texte, ITM ou WABA).
- `conversation_id`: (Optionnel) ID d'une conversation existante ou un déclencheur d'intégration :
  - `message-out` : Intégration immédiate dès l'envoi.
  - `replay-button` : Intégration après un clic sur un bouton.
  - `any-reply` : Intégration à la première réponse de l'utilisateur.
  - `conversed` : Intégration après engagement multi-messages.
- `conversation_tags`: (Optionnel) Tableau de tags à appliquer à la conversation générée.
- `recipients`: Liste des destinataires avec leurs données spécifiques.
- `global_data`: (Optionnel) Données partagées ou mapping dynamique.

## 2. Planification Avancée

Contrôlez précisément quand et comment votre campagne est envoyée en utilisant les champs de date spécialisés.

| Champ | Description | Réaction Client Post-Campagne |
| :--- | :--- | :--- |
| `start_datetime` | Date/heure la plus tôt pour commencer l'envoi. | N/A |
| `end_datetime` | (Bientôt disponible) Arrête l'envoi de **nouveaux** messages, mais les **réactions aux feedbacks clients** (boutons, flows) restent actives. | **Active** |
| `terminate_datetime` | (Bientôt disponible) Arrêt complet de la campagne **ET de toutes les réactions interactives**. Plus aucun traitement. | **Inactive** |

### Pourquoi utiliser à la fois `end` et `terminate` ?
- **`end_datetime`** : Arrêt professionnel de la diffusion. Utilisez ceci pour arrêter de créer de nouveaux fils de discussion tout en permettant aux utilisateurs ayant reçu le message d'interagir avec les boutons ou les flows.
- **`terminate_datetime`** : "Kill-switch" final. Utilisez ceci pour les **offres critiques dans le temps**. Une fois atteint, la campagne est totalement abandonnée, et tout clic sur un bouton ne déclenchera aucune réaction.

## 4. Mapping Dynamique avec `global_data`

Le bloc `global_data` vous permet d'éviter la répétition de données ou de créer des liens complexes en mappant des variables globales aux attributs des destinataires via la syntaxe `%recipient.champ%`.

### Cas d'Usage : Liens de Suivi Personnalisés

Plutôt que d'envoyer l'URL complète pour chaque destinataire, vous pouvez définir un pattern global :

```json
{
  "global_data": { 
    "company_name": "Digishare", 
    "tracking_link": "https://commandes.ma/%recipient.order_id%" 
  },
  "recipients": [
    { "wa_id": "2126...", "name": "John", "order_id": "123" },
    { "wa_id": "2126...", "name": "Jane", "order_id": "456" }
  ]
}
```

> [!TIP]
> Dans cet exemple, the first recipient will receive a link pointing to `https://commandes.ma/123`.

## 5. IDs et Tags de Conversation

Organisez automatiquement vos fils de discussion métier en liant les campagnes à des conversations existantes ou en appliquant des tags de classification.

```json
{
  "title": "Campagne Lead Gen",
  "conversation_id": "conv_987654321",
  "conversation_tags": ["nouveau_lead", "promo_été"],
  "channel": "whatsapp",
  "recipients": [ 
    { "wa_id": "212600000000", "name": "John" }
  ]
}
```

::important
Les tags sont appliqués à la conversation **dès que l'utilisateur répond** au message initial de la campagne ou dès que le message est distribué (selon vos paramètres de canal).
::
