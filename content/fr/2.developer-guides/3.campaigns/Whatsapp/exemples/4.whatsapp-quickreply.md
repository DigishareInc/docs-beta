Ce guide montre comment orchestrer des campagnes WhatsApp incluant des boutons de **Réponse Rapide (Quick Reply)** pour une interaction client fluide.

::note
**Prérequis** :
- Un **Bearer Token** valide dans l'en-tête Authorization.
- Un **Modèle de Message Digishare** (ITM ou **WABA TM**) configuré avec des boutons interactifs.
- Un **ID de Type de Message** pour l'orchestration de réponse rapide.
::

## 1. Logique Opérationnelle

Les boutons de réponse rapide permettent aux utilisateurs d'interagir avec votre marque en un clic. Chaque interaction génère un événement en temps réel capturé via l'[Orchestration de Webhooks](/fr/developer-guides/webhooks).

### Exigences de la Charge Utile
- `wa_id` : Numéro de téléphone du destinataire (dans `global_data` comme variable de modèle).
- `message_type_id` : Identifiant unique de la variante de modèle de réponse rapide.
- `tags` : Tableau optionnel pour catégoriser les campagnes.
- `conversation_id` : Identifiant optionnel pour le suivi des conversations.

## 2. Implémentation API

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Envoyez une campagne avec une orchestration interactive Quick Reply.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_ID_EXPEDITEUR
  templateId: VOTRE_ID_MODELE
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Question Enquête
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  message_type_id: "{messageTypeId}"
  global_data:
    wa_id: "%recipient.wa_id%"
  recipients:
    - wa_id: "212600000000"
  tags: []
  conversation_id: "message-out"
---
::

::tip
Les boutons interactifs améliorent considérablement les taux d'engagement pour les sondages, confirmations de RDV et menus de service.
::
