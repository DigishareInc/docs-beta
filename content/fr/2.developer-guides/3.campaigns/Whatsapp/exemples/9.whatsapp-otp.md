Ce guide démontre l'orchestration de campagnes WhatsApp incluant des codes **OTP (One-Time Password)** pour une authentification secondaire sécurisée.

::note
**Prérequis** :
- Un **Bearer Token** valide dans l'en-tête Authorization.
- Un **Modèle de Message Digishare** (**WABA TM**) approuvé dans la catégorie **Authentification**.
::

## 1. Gouvernance OTP

Les modèles OTP officiels supportent des fonctionnalités natives comme l'auto-complétion et les boutons "copier le code". Le champ `otp_code` est réservé à cet usage.

## 2. Implémentation API

Même pour les champs réservés, les valeurs dynamiques doivent être explicitement pontées via le bloc `global_data` pour chaque destinataire.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Envoyez une campagne de vérification OTP haute sécurité.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_ID_EXPEDITEUR
  templateId: VOTRE_ID_MODELE
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Verification OTP
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    otp_code: "%recipient.otp_code%"
  recipients:
    - wa_id: "212600000000"
      otp_code: "456789"
  tags: []
  conversation_id: "message-out"
---
::

::important
Le champ `otp_code` est réservé aux modèles **Authentification** WABA TM officiels. Pour les modèles texte standard, utilisez la logique de mapping de variable généralisée (ex: `{{1}}`).
::
