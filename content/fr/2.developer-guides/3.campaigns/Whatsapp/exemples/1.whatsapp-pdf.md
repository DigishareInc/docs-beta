Ce guide explique comment envoyer une campagne WhatsApp incluant un **document PDF**.

::note
**Prérequis** :
- Un **Bearer Token** valide.
- Un **Modèle de Message Digishare** (ITM ou **WABA TM**) configuré pour accepter un en-tête PDF.
::

## 1. Exigences de la Charge Utile

Pour envoyer un PDF personnalisé, vous devez ponter les variables `header_url` et `filename` via le bloc `global_data`.

### Logique Requise
- `wa_id` : Numéro de téléphone du destinataire.
- `header_url` : URL publique et accessible du PDF.
- `filename` : Le nom d'affichage du fichier sur WhatsApp.

## 2. Implémentation API

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Envoyez un document PDF personnalisé à plusieurs destinataires.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_SENDER_ID
  templateId: VOTRE_ID_MODELE
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Distribution de Factures
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    header_url: "%recipient.header_url%"
    filename: "%recipient.filename%"
  recipients:
    - wa_id: "212600000000"
      header_url: "https://votre-site.com/fact_janvier.pdf"
      filename: "facture_janvier.pdf"
    - wa_id: "212600000001"
      header_url: "https://votre-site.com/fact_fevrier.pdf"
      filename: "facture_fevrier.pdf"
---
::

## 3. Gestion des PDF Statiques

Si votre campagne utilise le même PDF pour tous les destinataires, définissez les valeurs une seule fois dans le bloc `global_data` :

```json
{
  "title": "Catalogue Mensuel",
  "global_data": { 
    "header_url": "https://votre-site.com/catalogue.pdf", 
    "filename": "Catalogue_Produits.pdf" 
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ]
}
```
