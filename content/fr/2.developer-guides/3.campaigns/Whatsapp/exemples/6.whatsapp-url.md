Ce guide détaille l'orchestration de campagnes WhatsApp présentant des boutons **Lien Externe** pour une navigation fluide vers vos propriétés web.

::note
**Prérequis** :
- Un **Bearer Token** valide.
- Un **Modèle de Message Digishare** (ITM ou **WABA TM**) contenant un bouton de type `URL`.
::

## 1. Gouvernance et Architecture des Liens

Digishare supporte deux architectures d'URLs au sein de vos modèles :
- **Statique** : Une URL de destination fixe commune à tous les destinataires.
- **Dynamique** : Une URL de base complétée par un suffixe variable (ex: `https://site.ma/commande/{{button0}}`).

## 2. Implémentation API

Pour l'orchestration d'URLs dynamiques, le suffixe variable doit être explicitement ponté via le bloc `global_data`.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Envoyez une campagne avec des suffixes de redirection URL dynamiques.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_ID_EXPEDITEUR
  templateId: VOTRE_ID_MODELE
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Paiement Commande
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    button0: "%recipient.button0%"
  recipients:
    - wa_id: "212600000000"
      button0: "ORD-789"
---
::
