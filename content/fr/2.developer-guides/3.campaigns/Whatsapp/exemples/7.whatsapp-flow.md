Ce guide démontre l'orchestration de campagnes WhatsApp incluant des **Flows** pour la collecte de données structurées et des parcours de service interactifs.

::note
**Prérequis** :
- Un **Bearer Token** valide.
- Un **Modèle de Message Digishare** (**WABA TM**) configuré avec un Meta Flow actif.
- Un **ID de Flow** et un **Start Screen ID** paramétrés dans votre Dashboard.
::

## 1. Logique Opérationnelle des Flows

Les Flows vous permettent de déployer des formulaires structurés directement dans l'interface WhatsApp. Les paramètres initiaux peuvent être passés au Flow via les champs `flow_token` et `flow_action_payload`.

## 2. Implémentation API

Les paramètres de Flow personnalisés doivent être pontés via le bloc `global_data` pour garantir la disponibilité des données session.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Envoyez une campagne présentant un WhatsApp Flow interactif.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_ID_EXPEDITEUR
  templateId: VOTRE_ID_MODELE
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Réservation Table
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    flow_token: "%recipient.flow_token%"
    flow_action_payload: "%recipient.flow_action_payload%"
  recipients:
    - wa_id: "212600000000"
      flow_token: "token_unique_123"
      flow_action_payload:
        screen: "START_SCREEN"
        data:
          init_val: "123"
---
::

::important
Les WhatsApp Flows sont une fonctionnalité avancée restreinte aux modèles **WABA TM** officiels. Assurez-vous que votre blueprint est approuvé par Meta avant le déploiement.
::
