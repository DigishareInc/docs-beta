## 1. Modèles de Message Digishare (Types)

Digishare supporte trois types de modèles pour vos campagnes :
- **Texte** : Simple message texte sans balises complexes.
- **ITM** : Interactive Template Message (boutons simples, listes).
- **WABA TM** : Modèles approuvés par Meta pour le marketing et les notifications.

::tip
Pour les campagnes à grand volume et le marketing, utilisez de préférence les modèles **WABA TM** pour garantir un taux de délivrabilité maximal et respecter les règles de Meta.
::

## 2. Variables Dynamiques

Placez les variables partagées dans `global_data` et les valeurs uniques dans chaque objet `recipients`.

- **Mappage Dynamique** : Utilisez `%recipient.variableName%` dans `global_data` pour générer des attributs complexes à partir des données individuelles des destinataires.
- **Cohérence** : Assurez-vous que chaque variable définie dans le modèle (ex: `{{1}}`, `{{name}}`) a une valeur correspondante.

## 3. Limites de Débit (Rate Limits)

::warning
Respectez les limites d'envoi pour éviter le blocage de votre numéro par WhatsApp.
::

- **Scaling** : Commencez avec de faibles volumes pour construire votre réputation d'envoi (Meta Tiering).
- **Efficacité** : Digishare gère la mise en file d'attente, mais vous devriez privilégier l'envoi groupé via le tableau `recipients` (jusqu'à 1000 destinataires par appel) plutôt que des appels individuels répétés.

## 4. Maintenance des Modèles

- **Approbation** : Seuls les modèles au statut `APPROVED` ou finalisés dans le dashboard peuvent être envoyés.
- **Modifications** : Toute modification d'un modèle **WABA TM** sur Meta nécessite une nouvelle période d'approbation avant de pouvoir être utilisée via l'API Digishare.
