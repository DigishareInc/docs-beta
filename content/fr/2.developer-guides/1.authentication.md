L'accès sécurisé à l'API Digishare est obligatoire. Nous prenons en charge deux méthodes principales d'authentification selon vos besoins d'intégration. Toutes les requêtes API doivent être effectuées via **HTTPS** ; les appels effectués en HTTP simple échoueront.

::alert{type="info"}
La meilleure façon de s'authentifier dépend de votre cas d'utilisation. Utilisez les **Jetons (PAT)** pour les scripts simples et **OAuth 2.0** pour les applications de production.
::

## 1. Jeton d'Accès Personnel (PAT)

Les Jetons d'Accès Personnels sont essentiellement des mots de passe pour votre compte que vous pouvez utiliser pour vous authentifier auprès de l'API. Ils sont idéaux pour les intégrations rapides ou lorsque vous n'avez pas besoin d'implémenter un flux OAuth complet.

::steps

### Générez votre Jeton

  Accédez à **Paramètres Avancés > API Credentials** ou utilisez le lien direct : [https://app.digishare.ma/api-credentials](https://app.digishare.ma/api-credentials).

### Sélectionnez les Scopes

  Choisissez les permissions spécifiques que ce jeton doit avoir. Nous recommandons le "Principe du Moindre Privilège" — n'accordez que ce dont vous avez réellement besoin.

### Copiez et Sécurisez

  Copiez le jeton immédiatement. Pour des raisons de sécurité, il ne sera **plus** affiché. Si vous le perdez, vous devrez le révoquer et en générer un nouveau.
::

### Exemple d'En-tête d'Autorisation

Incluez le jeton dans l'en-tête `Authorization` en tant que jeton `Bearer`.

::code-group

  ```bash [cURL]
  curl -X GET "https://api.digishare.ma/v1/user/profile" \
    -H "Authorization: Bearer VOTRE_JETON_ICI" \
    -H "Accept: application/json"
  ```

  ```javascript [Axios]
  import axios from 'axios';
  
  const response = await axios.get('https://api.digishare.ma/v1/user/profile', {
    headers: {
      'Authorization': 'Bearer VOTRE_JETON_ICI'
    }
  });
  ```

  ```javascript [Fetch]
  const response = await fetch('https://api.digishare.ma/v1/user/profile', {
    headers: {
      'Authorization': 'Bearer VOTRE_JETON_ICI',
      'Accept': 'application/json'
    }
  });
  const data = await response.json();
  ```

## 2. Identifiants Client OAuth 2.0

Utilisez le **Flux d'Identifiants Client** (Client Credentials Grant) pour la communication application-à-application où aucune intervention de l'utilisateur n'est requise. C'est la méthode la plus sécurisée pour les applications de production.

### Aperçu du Flux

1. **Échangez** votre `Client ID` et `Client Secret` contre un jeton d'accès.
2. **Récupérez** le jeton dans la réponse.
3. **Utilisez** le jeton dans l'en-tête `Authorization` pour tous les appels API.

### Demande de Jeton

Envoyez une requête `POST` au point de terminaison `/oauth/token`.

::code-group

  ```bash [Requête]
  curl -X POST "https://api.digishare.ma/v1/oauth/token" \
    -H "Content-Type: application/json" \
    -d '{
      "grant_type": "client_credentials",
      "client_id": "VOTRE_CLIENT_ID",
      "client_secret": "VOTRE_CLIENT_SECRET"
     }'
  ```

  ```json [Réponse]
  {
    "token_type": "Bearer",
    "expires_in": 31536000,
    "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiIsIm..."
  }
  ```

## Sécurité et Bonnes Pratiques

::alert{type="warning"}
**N'exposez jamais vos identifiants** dans le code côté client (navigateurs/mobile). Ceux-ci ne doivent être manipulés que dans des environnements serveur sécurisés.
::

* **Rotation** : Renouvelez régulièrement vos Jetons d'Accès Personnels et vos Secrets Client.
* **Révocation** : Si un jeton est compromis, révoquez-le immédiatement via le tableau de bord.
* **Scopes** : Limitez toujours la portée des jetons aux ressources spécifiques dont ils ont besoin.
* **Limites de Débit** : Toutes les méthodes d'authentification sont soumises à nos limites de débit API.
