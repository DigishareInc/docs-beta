Ce document explique les fonctionnalités principales pour gérer et interagir avec les conversations via l'API. Il décrit comment vérifier l'existence des conversations et comment leur envoyer des messages.

## 1. Vérifier une Conversation

Interrogez le système pour voir si une conversation existe déjà pour des paramètres spécifiques. Cela permet de récupérer l'état actuel d'une conversation et d'éviter les doublons.

**Endpoint** : `GET https://calclytics.digishare.ma/api/conversations/check`

### Paramètres de la Requête

Tous les identifiants (ID) doivent être fournis dans leur format encodé.

| Paramètre           | Type   | Description                                           |
| :------------------ | :----- | :---------------------------------------------------- |
| `company_id`        | String | L'ID encodé de votre entreprise.                      |
| `api_provider_id`   | String | L'ID encodé de l'instance API du fournisseur.         |
| `channel`           | String | Le canal de messagerie (ex : `whatsapp`, `sms`).      |
| `third_id`          | String | L'ID encodé du Contact (Third).                       |
| `channel_identity`  | String | L'identité associée au canal.                         |
| `platform_link_id`  | String | L'ID encodé du PlatformLink *(Optionnel)*.            |

### Exemple de Requête

```bash
curl 'https://calclytics.digishare.ma/api/conversations/check?company_id=780ew4kx3kdyjpa5&api_provider_id=zryw5xlrr8l06bqa&channel=whatsapp&third_id=0qwylwge5bvmpz65&channel_identity=212777786877' \
  -H 'sec-ch-ua-platform: "Windows"' \
  -H 'Authorization: Bearer <JWT_TOKEN>' \
  -H 'Referer: https://app.digishare.ma/' \
  -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)' \
  -H 'Accept: application/json, text/plain, */*' \
  -H 'sec-ch-ua-mobile: ?0'
```

### Réponse

Renvoie un objet JSON détaillant la découverte :

```json
{
  "success": true,
  "found": true, 
  "conversation": {
    "id": "...",
    "is_active": true,
    "last_message": { ... },
    "created_at": "..."
  },
  "identity_owner": "...",
  "timestamp": "2026-03-03T12:00:00Z"
}
```

## 2. Envoyer un Message

Envoyez un message d'un Agent (utilisateur du système) au contact dans une conversation.

**Endpoint** : `POST https://api-app.digishare.ma/v1/event/conversation_message`

### Paramètres Principaux / Payload

| Champ                      | Type    | Description                                                                              |
| :------------------------- | :------ | :--------------------------------------------------------------------------------------- |
| `event_type`               | String  | Doit être défini sur `conversation_message`.                                             |
| `channel`                  | String  | Le canal de messagerie (ex : `whatsapp`).                                                |
| `type`                     | String  | Le type de message (ex : `text`).                                                        |
| `send_to_third`            | Boolean | À définir sur `true` pour envoyer le message au fournisseur de la plateforme externe.    |
| `api_provider_instance_id` | String  | L'ID requis de l'instance du fournisseur de l'API.                                       |
| `recipient_id`             | String  | L'identifiant spécifique du destinataire.                                                |
| `body`                     | String  | Le contenu réel du message.                                                              |
| `conversation_id`          | String  | ID unique de la conversation *(Optionnel)*.                                              |

::warning
**Protection des 24 Heures** : Si le temps écoulé depuis le dernier message entrant dépasse 24 heures (ex : la fenêtre du service client sur WhatsApp), l'envoi de messages peut être restreint par votre fournisseur.
::

### Exemple de Requête

```bash
curl --location 'https://api-app.digishare.ma/v1/event/conversation_message' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <JWT_TOKEN>' \
--data '{
    "event_type": "conversation_message",
    "channel": "whatsapp",
    "type": "text",
    "send_to_third": true,
    "api_provider_instance_id": "6z70eykz3zmb84qo",
    "recipient_id": "212653565591",
    "body": "Message Body"
  }'
```

### Réponse (Succès)

```json
{
    "event_id": "2633736349319958528",
    "status": "accepted",
    "timestamp": "2026-02-12T14:22:23.133414141Z"
}
```
