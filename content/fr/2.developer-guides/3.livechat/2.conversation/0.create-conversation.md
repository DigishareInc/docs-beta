Créez une nouvelle conversation avec un contact via un fournisseur API. Le système résout automatiquement l'identité du contact et gère les conversations actives ou archivées existantes selon les paramètres fournis.

**Endpoint** : `POST https://api.digishare.ma/v1/conversations/provider`

### Paramètres Principaux / Payload

| Paramètre                          | Type    | Requis | Description                                                                             |
| :--------------------------------- | :------ | :----- | :-------------------------------------------------------------------------------------- |
| `provider_instance_id`             | String  | Oui    | L'ID de l'instance du fournisseur API gérant le canal.                                  |
| `channel`                          | String  | Oui    | Le canal de messagerie (ex : `whatsapp`, `sms`).                                        |
| `recipient_id`                     | String  | Oui    | Le numéro de téléphone, l'ID WhatsApp ou l'identifiant spécifique au canal.             |
| `platform_link_id`                 | String  | Non    | ID d'un PlatformLink existant. Ignore la résolution du contact si fourni.               |
| `archive_active_conversation`      | Boolean | Non    | Défaut : `true`. Archive la conversation active avant d'en créer une nouvelle.          |
| `reactivate_archived_conversation` | Boolean | Non    | Défaut : `false`. Réactive la dernière conversation archivée.                           |

### Exemple de Création

```bash
curl --location 'https://api.digishare.ma/v1/conversations/provider' \
  -H 'accept: application/json, text/plain, */*' \
  -H 'authorization: Bearer <JWT_TOKEN>' \
  -H 'content-type: application/json' \
  -H 'selectedcompanyid: <COMPANY_ID>' \
  --data-raw '{
    "provider_instance_id": "<PROVIDER_INSTANCE_ID>",
    "channel": "whatsapp",
    "archive_active_conversation": true,
    "reactivate_archived_conversation": false,
    "recipient_id": "<RECIPIENT_PHONE_NUMBER>",
    "third_id": "<THIRD_ID>"
  }'
```

### Déroulement

1. **Validation** : Vérifie que `provider_instance_id` existe.
2. **Résolution du Contact** : Résout le `PlatformLink`. S'il n'existe pas, crée un enregistrement `Third` minimal et un `PlatformLink` à la volée.
3. **Vérification de Réactivation** : Si `reactivate_archived_conversation` est `true`, la dernière conversation archivée est réactivée. L'active est d'abord archivée si `archive_active_conversation` est aussi `true`.
4. **Vérification d'Archivage** : Si une conversation active existe et que la réactivation n'est pas demandée, elle est archivée quand `archive_active_conversation` est `true`.
5. **Création** : Déclenche `CreateConversationAction` pour créer une nouvelle conversation active avec le template et le contact résolus.
