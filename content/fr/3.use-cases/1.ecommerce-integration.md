Ce document décrit le flux de travail pour l'intégration de votre **application e-commerce** avec la plateforme Digishare. Cela permet un support client automatisé et des notifications de commande proactives.

::note
**Prérequis** : Assurez-vous d'avoir authentifié votre client. Voir [Authentification](/developer-guides/authentication).
::

## 1. Cycle de Vie de l'Automatisation

::steps

### Préparation des Médias

(Optionnel) Téléchargez les documents ou médias nécessaires (ex: factures, images produits) pour obtenir des IDs persistants.

### Création du Ticket

Votre application e-commerce crée un ticket lorsqu'un événement de commande survient ou qu'un client demande de l'aide.

### Envoi de Message Automatisé

La plateforme Digishare déclenche automatiquement une notification WhatsApp personnalisée basée sur vos modèles prédéfinis.

### Synchronisation en Temps Réel

Digishare se synchronise avec votre application via des webhooks pour fournir des mises à jour en direct sur la distribution et l'engagement.
::

## 2. Préparation des Médias

**Point de terminaison** : `POST https://api.digishare.ma/v1/messagesTemplates`

Avant de créer un ticket avec une pièce jointe (comme une facture PDF), vous devez télécharger le fichier vers Digishare pour obtenir un `media_id`.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/messagesTemplates
description: Téléchargez un fichier (PDF, image) pour obtenir un media_id à utiliser dans les tickets.
variables:
  token: VOTRE_TOKEN
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  type: file
  category: document
  file:
    base64: "data:application/pdf;base64,JVBERi0xLjQKJ..."
    file_name: facture
    mime_type: application/pdf
    extension: pdf
---
::

> **Réponse** : Enregistrez le `data.id` retourné (ex: `YOUR_MEDIA_ID`). Utilisez-le comme `PRE_UPLOADED_HASHED_ID` dans l'étape suivante.

---

## 3. Orchestration des Tickets et Campagnes

**Point de terminaison** : `POST https://api.digishare.ma/v1/tickets`

Initiez le flux de support ou de notification en créant un ticket. Vous pouvez mapper des références externes (numéros de commande) pour lier vos données système à Digishare.

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/tickets
description: Créez un ticket de support pour la confirmation de commande ou les problèmes clients.
variables:
  token: VOTRE_TOKEN
  mediaId: PRE_UPLOADED_HASHED_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  external_id: "COMMANDE-12345"
  type_ticket_id: npvrxamq9l94y8dz
  ticket_status_id: x9q3wrlv0m7vzea8
  create_third_data: true
  information:
    third:
      ext_ref: "CLIENT-67890"
      name: Ahmed Alami
      wa_id: "212600000000"
      email: ahmed.alami@example.com
    campaign_data:
      order_id: "12345"
      invoice_document:
        provider: digishare
        id: "{mediaId}"
      items_count: "5"
      order_total: "500.00"
      delivery_address: "Casablanca, Bd Zerktouni"
  subject: Confirmation de Commande
  lang: fr
---
::

::tip
Utilisez `campaign_data` pour passer des variables dynamiques utilisables dans vos modèles WhatsApp.
::

## 4. Synchronisation en Temps Réel

Configurez votre système pour écouter les événements `ticket.updated`.

**Événement** : `ticket.updated`

```json
{
  "event": "ticket.updated",
  "data": {
    "id": "hashed_ticket_id",
    "data": {
      "id": "hashed_ticket_id",
      "external_id": "COMMANDE-12345",
      "subject": "Confirmation de Commande",
      "ticket_number": "342106",
      "type_ticket_id": "npvrxamq9l94y8dz",
      "third_id": "hashed_third_id",
      "ticket_status_id": "yqdwrbka7kz69v8e",
      "information": {
        "third": {
          "name": "Ahmed Alami",
          "email": "ahmed.alami@example.com",
          "wa_id": "212600000000",
          "ext_ref": "CLIENT-67890"
        },
        "campaign_data": {
          "order_id": "12345"
        }
      }
    },
    "changes": {
      "ticket_status_id": "yqdwrbka7kz69v8e"
    }
  }
}
```

::tip
Le `ticket_status_id`: `yqdwrbka7kz69v8e` correspond au statut **Commande Confirmée**.
::
